# 木棒延伸检测与误差评估设计文档

**日期**: 2026-04-29
**主题**: 基于背景差分与颜色过滤的木棒延伸量检测算法

---

## 1. 问题定义

### 1.1 输入数据
- **1面/2面**: 基坑两个侧面，分别拍摄
- **XmYmmZ.bmp**: 相机距墙面 Xm，木棒向外延伸 Ymm，第 Z 张图
- **empty.bmp**: 无木棒的空场景（背景模板）
- **ruler.bmp**: 1m 标尺置于 10m 处（像素↔物理比例标定）
- **标定参数**: OpenCV 相机内参矩阵 K 与畸变系数 D

### 1.2 输出
- 每张图检测到的木棒物理延伸量（mm）
- 与真实值对比的绝对误差、相对误差
- 按距离（10m vs 18m）和按位移量分组的误差统计
- 可视化图表（误差-距离散点图、误差-位移柱状图）

### 1.3 成功标准
- 检测算法在 10m 距离下平均绝对误差 ≤ 2mm
- 检测算法在 18m 距离下平均绝对误差 ≤ 5mm
- 输出完整的误差评估 CSV 和可视化图表

---

## 2. 核心检测流程

```
输入：待测图 + 对应 empty 图 + ruler 图 + 标定参数
     ↓
[1] 去畸变（undistort）
     ↓
[2] 背景差分（待测图 - empty 图）
     ↓
[3] HSV 颜色阈值过滤（木棕色范围）
     ↓
[4] 形态学去噪（开运算 + 闭运算）
     ↓
[5] 轮廓检测 → 最大连通区域 = 木棒
     ↓
[6] 检测木棒最外侧端点
     - 1面：取最左侧像素（木棒向左伸出）
     - 2面：取最右侧像素（木棒向右伸出）
     ↓
[7] 像素延伸量 × ruler 标定比例系数 → 物理延伸量（mm）
     ↓
[8] 与真实值对比，计算误差
```

---

## 3. 关键子模块设计

### 3.1 去畸变
- 使用标定参数 `camera_matrix` 与 `dist_coeffs`
- 对每张输入图调用 `cv2.undistort()`
- 确保后续像素测量在直线性校正的图像上进行

### 3.2 像素↔物理比例标定（ruler 处理）
- **输入**: `ruler` 图（1m 标尺在 10m 处）
- **方法**:
  1. 灰度化 + 边缘检测（Canny）
  2. Hough 直线检测，找出标尺的两条长边
  3. 计算两长边之间的像素距离 `L_px`
  4. 比例系数 `k = 1000mm / L_px`（mm/px）
- **输出**: `k`（用于 10m 距离图的比例换算）
- **注意**: 18m 距离图的比例系数按比例推算：`k_18m = k_10m × (10/18)`

### 3.3 背景差分
- **输入**: 去畸变后的待测图 + 去畸变后的 empty 图
- **方法**:
  1. 两图均转灰度
  2. 绝对差分：`diff = |img - empty|`
  3. 二值化：`mask = diff > threshold`
  4. threshold 默认 20，可通过直方图自适应调整
- **输出**: 二值差分掩码

### 3.4 颜色过滤
- **输入**: 原始待测图（去畸变后）
- **HSV 阈值范围**（木棕色，经验值，需现场调参）:
  - H: [10, 30]
  - S: [40, 255]
  - V: [20, 255]
- **方法**:
  1. 转 HSV
  2. `cv2.inRange()` 生成颜色掩码
  3. 与差分掩码做逻辑与：`final_mask = diff_mask & color_mask`
- **输出**: 最终候选掩码

### 3.5 形态学去噪与连通区域提取
- **开运算**（先腐蚀后膨胀）：去除细小噪点
- **闭运算**（先膨胀后腐蚀）：填补木棒内部空洞
- 核大小: 3×3 或 5×5，根据实际效果调整
- 提取所有连通区域，取面积最大的作为木棒

### 3.6 端点检测与延伸量计算
- **基准零点**: empty 图中平台边缘的像素位置
  - 1面: 平台右边缘 x 坐标（木棒从右向左伸）
  - 2面: 平台左边缘 x 坐标（木棒从左向右伸）
  - 检测方式：分析 empty 图的垂直边缘，结合 ROI 区域定位
- **端点坐标**: 木棒连通区域的最外侧像素
  - 1面: `rod_tip_x = min(x for x in rod_region)`
  - 2面: `rod_tip_x = max(x for x in rod_region)`
- **像素延伸量**: `|rod_tip_x - baseline_x|`
- **物理延伸量**: `pixel_extension × k`（mm）

### 3.7 误差计算
- **绝对误差**: `|detected_mm - ground_truth_mm|`
- **相对误差**: `absolute_error / ground_truth_mm × 100%`（对 0mm 样本用绝对误差）
- **分组统计**:
  - 按距离: 10m 组 vs 18m 组 → 均值、标准差、最大/最小
  - 按位移: 0mm / 5mm / 10mm / 20mm 组

## 5. 输入输出规格

### 5.1 输入
- 图片文件夹路径: `1面/`, `2面/`
- 标定文件路径: `标定/calibration_result.yaml`
- 配置参数（可选，均有默认值）:
  - HSV 范围
  - 差分阈值
  - 形态学核大小
  - ROI 区域（限制检测范围，加速并减少误检）

### 5.2 输出
- **CSV 文件**: 每行一张图
  - `filename, face, distance_m, ground_truth_mm, detected_mm, abs_error_mm, rel_error_pct`
- **文本汇总**: 按距离和位移分组的统计量
- **图表**:
  - 误差-距离散点图（10m vs 18m 分布）
  - 误差-位移量柱状图（各真实位移下的平均误差）

## 6. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 光照变化导致颜色阈值失效 | 差分/颜色过滤效果差 | 提供交互式调参工具；记录当前最优参数 |
| 木棒与背景颜色相近 | 检测不到完整木棒 | 增强背景差分权重；调整 HSV 范围 |
| 0mm 样本木棒不可见 | 无法检测基准 | 直接用 empty 图作为 0mm 参考；误差计算用 0 作为真实值 |
| ruler 检测不准 | 比例系数 k 偏差 | 提供手动标定模式；多 ruler 图取平均 |
| 18m 图像分辨率不足 | 端点检测抖动大 | 增大形态学核；使用亚像素级角点检测 |

---

## 7. 扩展性

- 所有参数外置为配置变量，方便调整
- 检测流程模块化，便于替换为其他算法（如深度学习）
- 误差评估模块独立，可与任何检测算法对接
